🤖 Applying knowledge of @orchestrator & ECC + Ag-Kit (Hybrid)...

V 하이브리드 안티그래비트 킷트(ECC + Ag-Kit) 활용 가이드

두 가지 키트가 모두 설치된 현재, 상황에 맞는 최적의 명령어를 사용하여 저에게 작업 지시를 내리시면 됩니다. 다음은 용도와 필요에 따른 효과적인 지시 방법입니다.

🤖 [키트 1] ECC (Everything Claude Code) 활용
강점: 기술적 완벽성, 보안, 테스트 코드, 심층 아키텍처, 성능 최적화

기술적인 깊이가 필요하거나 코드의 안정성이 중요한 경우 다음과 같이 지시하세요.

지시 예시 1 (계획 수립): "현재 프로젝트의 아키텍처를 분석하고 /plan 명령어를 사용하여 신규 기능을 도입하기 위한 단계별 전략을 세워줘."
지시 예시 2 (코드 리뷰): "작성된 코드에 대해 /code-review와 /security-reviewer 스킬을 사용하여 보안 취약점과 클린 코드 준수 여부를 검토해줘."
지시 예시 3 (테스트): "신규 로직에 대해 /tdd 방식으로 접근하고, 최종적으로 /e2e 테스트를 수행해줘."


🤖 [키트 2] Ag-Kit (Antigravity Kit by vudovn) 활용
강점: UI/UX 디자인, 빠른 대시보드 구축, 프론트엔드 시각화, 기능 개선

사용자 화면의 아름다움이나 인터랙션, 빠른 대시보드 구축이 필요할 때 다음과 같이 지시하세요.

지시 예시 1 (UI 최적화): "현재 정산 모달의 디자인을 /ui-ux-pro-max 가이드라인에 맞춰 프리미엄 스타일로 개선해줘."
지시 예시 2 (기능 추가): "분석 대시보드에 새로운 차트를 추가하고 /enhance 명령어로 전체적인 완성도를 높여줘."
지시 예시 3 (수동 가이드): "공정 관리 페이지의 UX를 /status 명령어로 점검하고 개선할 포인트들을 /brainstorm 해줘."
💡 팁: 통합 및 지리적 명령 (Slash Command) 사용법
이제는 제가 두 키트의 모든 슬래시 커맨드를 이해하고 있으므로, 그냥 **"명령어"**만 직접 언급하셔도 됩니다.

복잡한 비즈니스 로직 수정 시: "안전하게 수정해야 하니 /plan 먼저 하고 시작하자."
화면 디자인 요청 시: "이 대시보드 화면을 /ui-ux-pro-max 스타일로 바꿔줘."
에러 해결 시: "발생한 버그를 /debug 모드로 철저히 조사해서 근본적인 원인을 고쳐줘."
V 결론: "기술적 무결성"이 필요할 땐 ECC(키트 1)를, "시각적 우수성 및 빠른 개선"이 필요할 땐 Ag-Kit(키트 2)를 선택하여 명령하시면 제가 그에 최적화된 스킬과 워크플로우를 즉시 적용합니다.

```javascript

 * 🚀 안티 그래비티 자동 클릭 + 터미널 자동 숨김 매크로(Accept , Always Allow , Accept all , Allow Once)

/**
 * 🚀 안티 그래비티 비침투성 자동 클릭 매크로
 * 특징: 터미널 강제 은폐(CSS Injection) + 즉각적 포커스 복구

///////////////

(function() {
    // 중복 실행 방지: 전역 플래그 확인
    if (window.antiGravityRunning) {
        console.log("[🛡️] 이미 스캔 중입니다.");
        return;
    }
    window.antiGravityRunning = true;

    const styleId = 'anti-gravity-stealth-css';
    if (!document.getElementById(styleId)) {
        const style = document.createElement('style');
        style.id = styleId;
        style.textContent = `
            .part.panel.bottom, .terminal-wrapper, [id="workbench.parts.panel"] { 
                display: none !important; visibility: hidden !important; 
            }
        `;
        document.head.appendChild(style);
    }

    // 전역 실행 잠금 (중복 클릭 방지용)
    let globalLock = false;

    function safeAutoClick(root) {
        if (globalLock) return;

        const selectors = 'button, a, [role="button"], .monaco-button, vscode-button';
        const elements = root.querySelectorAll(selectors);

        for (const el of elements) {
            if (el.shadowRoot) safeAutoClick(el.shadowRoot);
            
            const combinedText = ((el.textContent || "") + " " + (el.getAttribute('aria-label') || "")).toLowerCase();
            const isTarget = (combinedText.includes('run') && !combinedText.includes('debug')) || 
                             /allow once|accept|always allow/.test(combinedText);

            // 이미 처리 중이거나 잠긴 요소는 건너뜀
            if (isTarget && !el.dataset.processed) {
                globalLock = true;
                el.dataset.processed = "true"; // 락을 더 확실하게 선언
                
                console.log(`[🛡️] 실행 대상 확정: ${combinedText}`);
                
                // 즉시 클릭보다는 약간의 지연을 주어 셸 준비 시간 확보
                setTimeout(() => {
                    el.click();
                    
                    // 실행 후 포커스 복구 및 락 해제
                    setTimeout(() => {
                        globalLock = false;
                        // 한 번 처리된 버튼은 다시 클릭하지 않도록 유지 (필요시 삭제)
                        // delete el.dataset.processed; 
                    }, 2000); 
                }, 100);
                break; // 한 번에 하나의 액션만 수행하여 충돌 방지
            }
        }
    }

    const mainInterval = setInterval(() => {
        try {
            safeAutoClick(document);
        } catch (e) {
            console.error("[🛡️] 스캔 중 오류 발생:", e);
            globalLock = false;
        }
    }, 1500); // 간격을 약간 늘려 여유를 줌

    window.antiGravityMacro = mainInterval;
    console.log("✅ 개선된 스텔스 매크로가 가동되었습니다.");
})();
